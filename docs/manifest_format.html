AgentHub Manifest Format

YAML schema for registering agents in AgentHub

Required Fields

Field Type Description
namerequired string Unique agent name. Lowercase letters, numbers, and hyphens only. Must start and end with a letter or number. Max 100 characters.
versionrequired string Semantic version in format X.Y.Z (e.g., 1.0.0)
descriptionrequired string Brief description of what the agent does. Max 500 characters.
authorrequired string Author name or organization.

Optional Fields

Field Type Description
capabilitiesoptional list[string] List of capabilities this agent provides (e.g., code-review, text-generation)
protocolsoptional list[string] Supported communication protocols.
MCP A2A custom
permissionsoptional list[string] Required permissions (e.g., read-files, network-access, execute-code)
lifecycle_stateoptional string Current lifecycle state. Defaults to active.
active deprecated retired revoked

Complete Example

# Agent Manifest for AgentHub

name: code-reviewer
version: 1.0.0
description: An AI agent that reviews code for bugs and style issues
author: your-team

capabilities:
  - code-review
  - bug-detection
  - style-checking

protocols:
  - MCP

permissions:
  - read-files
  - network-access
